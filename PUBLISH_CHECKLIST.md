# Hanemy β v0.9.1.1 Public Beta 公開チェックリスト

## 1. ファイル構成

GitHub Pagesで公開する場合、以下のファイルがリポジトリ直下にあることを確認します。

- `index.html`
- `style.css`
- `app.js`
- `share.js`
- `manifest.json`
- `sw.js`
- `favicon.png`
- `icon-192.png`
- `icon-512.png`
- `README.md`
- `TEST_CASES.md`
- `PUBLISH_CHECKLIST.md`
- `FEEDBACK_GUIDE.md`
- `LICENSE`

## 2. PWA / キャッシュ

- `index.html` のCSS/JS/manifest参照が `v=090` になっている
- `manifest.json` の `start_url` と `id` が `v=090` になっている
- `sw.js` のキャッシュ名が `hanemy-beta-v0-9-public-beta` になっている
- 古い表示が残る場合は、ブラウザのサイトデータを削除して再確認する

## 3. スマホ確認

- iPhone Safariで表示確認
- Android Chromeで表示確認
- ホーム画面追加時のアイコン確認
- 下部ナビが画面下に隠れないか確認
- 入力ボトムシートが狭すぎないか確認

## 4. 金額ロジック

- `TEST_CASES.md` の1〜7を手入力で確認
- 生活費超過時に、累積貯蓄が勝手に減らないことを確認
- 全体超過時だけ、貯蓄補填ボタンが出ることを確認
- 月初更新で、使わなかった分が貯蓄に回ることを確認

## 5. 共有カード

- 親共有カードの画像が生成される
- 収入・固定費・カテゴリ内訳が表示されない
- 文言が重すぎない
- 画像として保存または共有できる

## 6. 公開後に見ること

公開後は、褒め言葉よりも「迷った場所」を集めます。

- 最初に何をすればいいか分かったか
- 今月あと使える金額はすぐ分かったか
- 入力は面倒ではなかったか
- 貯蓄の意味は分かったか
- 親に送るカードは自然だったか
- 毎日開きたい重さだったか

## 7. v0.9.1 の合格基準

次を満たせば、v1.0候補へ進めます。

- 初回起動から1分以内に設定できる
- 支出入力が数タップで終わる
- 金額ロジックにズレがない
- スマホで毎日開いても重くない
- 親共有カードが自然に見える
- 追加したい機能より、削るべき違和感の方が少ない
