# Hanemy / ハネミー

**Hanemy（ハネミー）** は、家計簿ではなく、  
**今月を安心して過ごすためのお金の作戦ボード**です。

細かい支出記録を目的にせず、  
「今月あとどれくらい使えるか」  
「生活費と楽しみ用がどれくらい残っているか」  
「使った分をざっくり反映する」  
ことに集中した、スマホ向けの軽量Webアプリです。

---

## 公開βバージョン

**Hanemy β / v0.29.6.4 direction choices cleanup**

---

## コンセプト

ハネミーは、一般的な家計簿アプリとは少し違います。

通常の家計簿アプリは、  
「何にいくら使ったか」を細かく記録することが中心です。

ハネミーは、そこではなく、

- 今月あとどれくらい使えるか
- 生活に必要なお金は守れているか
- 楽しみに使うお金を残せているか
- 今月やりたいことに向けて、無理なく進めているか

を、できるだけ少ない操作で確認できることを目指しています。

---

## 主な画面

### ホーム

ホームは、アプリの中心となる画面です。

表示される主な情報：

- 今月の残り
- 今日の目安
- 生活費の残り
- 楽しみ用の残り
- 残せそうな見込み
- 今月やりたいこと
- カテゴリ別の使用状況
- 共有機能
- 今月の準備への導線

ホームは、**今月の現在地を見る場所**です。

---

### ざっくり入力

使った金額を、細かく記録せずに反映する画面です。

できること：

- 大カテゴリを選ぶ
  - 生活
  - 楽しみ
  - やりたい
- 小カテゴリを選ぶ
- 金額ボタンで入力予定額を作る
- ＋使った / −取り消し を切り替える
- 入力履歴を確認する
- 入力後に取り消す

目的は、家計簿のように細かく記録することではなく、  
**今月の作戦ボードに、使った分だけざっくり反映すること**です。

---

### 今月の準備

月初や、収入・予定が変わった時に使う画面です。

設定する内容：

- 管理期間
- 今月の方針
- 今月やりたいこと
- 収入
- 必ず出ていくお金
- 生活費
- 楽しみ用のお金

毎日使う画面ではなく、  
**今月の作戦を最初に整える場所**です。

---

### カテゴリ別に見る

ホーム内の補助表示です。

表示内容：

- 生活費の全項目
- 楽しみ用の全項目
- 今月やりたいことの全項目
- 各項目の使用率
- 使った金額 / 目安金額
- 使用率バー
- 100%超過時の強調表示

細かい分析画面ではなく、  
**どこにどれくらい使っているかを一目で見るための一覧**です。

---

### 共有する

今月のお金の作戦を、画像として共有・保存できます。

共有タイプ：

#### 家族に共有

家族や保護者など、生活を心配してくれる人に向けて、  
「ちゃんと考えてお金を使っている」ことを伝えるための共有です。

表示内容：

- 今月の残り
- 今日の目安
- 生活費の残り
- 楽しみ用の残り
- 残せそうな見込み
- 今月やりたいこと
- ひとこと

#### 画像で保存

誰かに送る前提ではなく、  
自分用に今月の作戦を画像として残す機能です。

---

### お知らせ

右上のベルアイコンから開きます。

お知らせには、日付と公開時刻を表示します。

表示例：

```text
2026-04-28 00:25 更新
```

時刻は自動取得ではなく、  
お知らせデータ内に手動で設定する公開時刻です。

---

### 設定

右上の設定アイコンから開きます。

現在の内容：

- データ管理
- 確認用デモ
- フィードバック
- 保存について

設定は、画面遷移ではなくポップアップとして表示されます。

---

## 保存方式

データは、ブラウザの `localStorage` に保存されます。

そのため、

- ログイン不要
- サーバー不要
- 端末内で完結
- ブラウザデータを削除するとデータも消える

という仕様です。

## PWA対応

ハネミーは、PWAとしてホーム画面に追加できます。

使用ファイル：

- `manifest.json`
- `sw.js`
- `icon-192.png`
- `icon-512.png`
- `favicon.png`

Service Worker によりキャッシュされるため、更新時にはキャッシュ名を変更します。

現在のキャッシュ名：

```js
hanemy-assets-v0292-brand-assets
```

```text

icon-192.png
icon-512.png
index.html
manifest.json
sw.js
```

---

## 削除してよいファイル

公開用リポジトリでは、以下は不要です。

```text
.DS_Store
README.txt
logo.png
CLEANUP_REPORT.json
PWA_POLICY.json
```

理由：

- `.DS_Store` はMacの自動生成ファイル
- `README.txt` は `README.md` と重複
- `logo.png` はルート直下では不要。最新仕様では `assets/logo.png` を使用
- `CLEANUP_REPORT.json` と `PWA_POLICY.json` は開発管理用で、公開アプリには不要

## ブランド表記

アプリ最下部には以下のブランド表記を表示しています。

```text
Hanemy™
```

`™` は、商標として使っていることを示す記号です。

`®` は登録商標を示す記号なので、  
商標登録が完了するまでは使用しない方針です。

登録完了後は、必要に応じて以下に差し替えます。

```text
Hanemy®
```

## 現在の開発方針

公開βでは、機能追加よりも以下を優先します。

- ホームが直感的に理解できるか
- ざっくり入力が迷わず使えるか
- 共有画像が自然に使えるか
- スマホで表示が崩れないか
- PWA更新が詰まらないか

## 公開前チェックリスト

公開前に確認する項目：

- ホームの作戦ボードがスマホで崩れていない
- ざっくり入力が1画面で使いやすい
- 入力後の「入力しました」が自動で消える
- カテゴリ別に見るで使用率が確認できる
- 共有する → 家族に共有 / 画像で保存 が動く
- お知らせに日付・時刻が表示される
- 設定の確認用デモが動く
- データ削除の確認ダイアログが出る
- ホーム画面追加後も更新が反映される

## v0.29 design system cleanup

README更新ルール：今回も変更内容を追記しています。今後の更新でも必ず追記します。

### 目的

機能追加ではなく、デザインの土台を整理する更新です。

ハネミーのデザイン方針は、

> かわいい家計簿ではなく、やさしい作戦ボード。

です。

### 変更内容

- 色のルールをCSS変数として整理
- 角丸のルールを整理
  - 大ボード：28px
  - カード：24px
  - 小パーツ：18px前後
  - チップ：999px
- 影のルールを整理
  - 大ボードはやわらかく浮かせる
  - カードは控えめ
  - ボタンは押せる程度に強調
- 文字サイズと階層を整理
- ホーム・ざっくり入力・ふり返り系ボードの見た目を統一
- ボタン、チップ、ポップアップ、お知らせの質感を統一
- READMEを現在仕様に合わせて更新

### デザイン比率

現段階のハネミーでは、以下を目安にします。

```text
70%：安心・見やすさ
20%：かわいさ
10%：遊び心
```

### 今後のデザイン調整方針

- 画像や装飾を増やす前に、余白・文字・ボタンの一貫性を優先
- ホームとざっくり入力を最重要画面として磨く
- 共有画像は、アプリ内UIとは別に「見せたくなる画像」として後から磨く
- 金融アプリの硬さではなく、生活アプリのやさしさを維持する
